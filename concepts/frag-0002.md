A koru library transform can **synthesize a whole callable event** — event decl
+ implementation — from a declaration, the way `store` synthesizes its
insert/query family. `vaxis:component(greet){ <text>{{ name:s }}</text> }` proves
it: the `{{ name:s }}` placeholders become the event's typed input shape, and the
markup becomes an impl flow that composes the existing `write-at`. `greet` is then
a normal koru event — callable, branchable, loopable. This extends [[frag-0001]]
(transforms lower to real invocations) up one level: transforms can also DEFINE
the events those invocations call.

Making it actually reach the binary required three things, each a real gotcha:

1. **Return `whole_program`, not per-site `.appended`.** The shape checker builds
   its event set by re-scanning `program.items`, so the synthesized event must BE
   in the program it scans. A whole-program walk (find every declaration,
   synthesize, consume the declarations, hand back the rewritten program) is the
   clean shape — a library transform holds the whole AST, so it can. Per-site
   `.appended` did not make the event visible.

2. **Run the collector in the `pre` transform stage.** Handler generation for an
   event's impl runs after `pre` but the synthesized event must exist before it —
   `[comptime|transform|pre]`.

3. **THE KILLER: stamp the consumer's module qualifier onto the synthesized
   event's path, or `dead_strip` prunes it.** dead_strip keeps an event only if
   its path-key appears in the invoked-set, keyed by `pathToString` (`module:seg`
   when a qualifier is present, bare `seg` otherwise). A normal event is stamped
   with its module by the `canonicalize_names` pass, and the bare call site is
   stamped to match — so both key as `<module>:name`. A SYNTHESIZED event is added
   AFTER canonicalize, so if its path carries a `null` qualifier it keys as bare
   `name`, the call keys as `<module>:name`, they don't match, and dead_strip
   silently strips the event right before emit. The fix: build the event (and its
   `impl_of`) path with `module_qualifier = <consumer module>` so it keys the same
   as the canonicalized call. General rule: **anything synthesized after
   canonicalize must self-apply what canonicalize would have done** — module
   qualifiers first among them.

Diagnosis lever that cracked #3: `KORU_DUMP_AST=1` dumps item counts per pipeline
stage. Seeing `post-transform items=22` → `emit-zig items=20` localized the loss to
the dead_strip pass between them, in one run — far faster than reading emitter code.
